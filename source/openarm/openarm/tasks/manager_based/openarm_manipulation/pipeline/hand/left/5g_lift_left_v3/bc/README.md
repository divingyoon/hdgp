# 5g_lift_left_v3 BC (t2 + Demo)

This folder provides a practical starting point for demo-based grasp BC.

## Files

- `retarget_allegro_to_tesollo.py`
  - Converts GraspXL-style Allegro demo sequence into Tesollo joint targets and v3 action vectors.
- `collect_v3_observations.py`
  - Replays action sequence in `5g_lift_left-v3` and records the actual policy observations.
- `retarget_config_example.json`
  - Limit / offset / scale / mapping template used by the converter.
- `train_bc_mlp.py`
  - Trains an MLP behavior cloning regressor from `(observations, actions)`.

## Output Dataset Format (`.npz`)

Generated by `retarget_allegro_to_tesollo.py`:

- `actions`: shape `[T, action_dim]` (default action_dim=32)
- `tesollo_joints`: shape `[T, 20]` (`lj_dg_1_1..5_4`)
- `synergy`: shape `[T]` (`left_hand_action` scalar)
- `thumb_action`: shape `[T, 8]` (`left_thumb_action`)
- `meta_json`: JSON string metadata

For BC training, add `observations` (same action-aligned time steps) to the NPZ.
If `observations` is missing, `train_bc_mlp.py` can fallback to `tesollo_joints`.

## Typical Workflow

1. Convert external demo
2. Build/align observations from your v3 rollout pipeline (or use `collect_v3_observations.py`)
3. Run BC pretrain
4. Use BC weights as initialization for PPO fine-tuning

## Example

```bash
python retarget_allegro_to_tesollo.py \
  --input /path/to/allegro_seq.npy \
  --config retarget_config_example.json \
  --output /tmp/demo_tesollo_v3.npz

python collect_v3_observations.py \
  --task 5g_lift_left-v3 \
  --input-actions /tmp/demo_tesollo_v3.npz \
  --output /tmp/demo_tesollo_v3_with_obs.npz \
  --headless

python train_bc_mlp.py \
  --dataset /tmp/demo_tesollo_v3_with_obs.npz \
  --out /tmp/bc_actor.pt
```
